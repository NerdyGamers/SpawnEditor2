Introduction  [Top]

Welcome to the Spawn Editor user manual.  This document will try to explain how to use the editor to quickly and accurately create spawners in RunUO.

[Top]

Overview  [Top]

The main screen of Spawn Editor is broken down into three areas.  The first area is the left panel which contains two groups of controls.  The top group called Map Settings is used for configuring the application as well as some map behaviour controls.  The bottom group is called Spawn Types and is used for selecting and displaying what type of objects the spawner is supposed to create.  The Spawn Types list can be changed to show all items and mobiles, only items, or only mobiles.  The bottom of the Spawn Types group has a total count of the types currently available in the list.  The second area of the application is the centre panel where the spawners can be edited visually.  The map can be zoomed in or out using the zoom slider found in the Map Settings area of the left panel.  The third area of the application is the right panel and contains two groups of controls.  The top group is called Spawn Details and is used to set the spawners properties.  The bottom group is called the Spawn List and shows all of the spawners currently defined.

The editor works based on the current version of RunUO and the scripts associated with it.  When the editor starts for the first time, it attempts to load all types defined in the RunUO server by loading the assemblies of the following binaries:

  • RunUO.exe
  • Scripts\Output\Scripts.dll
  • Scripts\Output\Scripts.CS.dll
  • Scripts\Output\Scripts.VB.dll

This reason for this is to populate the Spawn Types control located in the left panel of the editor.  Therefore, it is necessary to have compiled all RunUO scripts prior to starting the editor, otherwise no types will appear in the list.  The other requirement for the editor to work is that it must have access to any map*.mul files required to draw the map in the centre panel which leads us to the configuration of the application.

Installation  [Top]

  1. Run the Spawn Editor installer, Setup.msi.  This will install the editor and register the UOMap.ocx.
  2. Take the file SpawnEditorCommands.cs from the directory created in step 1 and place it into your RunUO\Scripts\Commands directory.
  3. Take the file XmlSpawner.cs from the directory created in step 1 and place it into your RunUo\Scripts\Engines\Spawner directory.
  4. Go to Start... Programs... Spawn Editor... and click on the SpawnEditor  shortcut to launch the editor.
  5. If this is the first time running Spawn Editor, you will be brought to the Configuration screen to specify the directories where RunUO and your Ultima Online client reside. Spawn Editor will attempt to automatically find your UO client based on the current registry settings for UO. If this is incorrect, simply change the directory to point to any valid UO map files.  See the Configuration section for more information on the items in the dialog.

    NOTE:  .NET applications have security restrictions imposed on them by default.  If you would like to run the Spawn Editor from a share or any mapped drive you will need to do the following:

    i.  Click on Start... Settings... Control Panel... Administrative Tools... Microsoft .NET Framework Wizards...

    ii. Click on Adjust .net Security

    iii. Select Make changes to this computer and click Next.

    iv. Click on Local Intranet and move the slider up to the top marker where it says Full Trust.

    v. Click Next and then click Finish.
  6. All Done! Happy Spawning!


    Note: An additional RunUO spawner script is included called XmlSpawner which is what Spawn Editor natively supports as a spawner. You don't have to use it, but it does have a slightly better gump than the standard spawner object and you can define a rectangle rather than a regular square spawn area with it. Spawn Editor was originally developed with XmlSpawner in mind. Just place XmlSpawner in RunUO\Scripts\Engines\Spawner folder and recompile as stated in step 4 above.

Configuration  [Top]

The editor needs a bit of configuration before it can work.  There are two key pieces of information that must be supplied to the editor for it to function properly.  These are:

  • Path to the RunUO executable
  • Path to the Ultima Online client executable

These items, along with a few other options can be configured in the Configuration dialog.  The Configuration dialog will appear the first time the application is started.  The configuration is stored on a per user basis in the current users registry key under the following path:

HKEY_CURRENT_USER\Software\Spawn Editor

All configuration information used by the editor is stored in this area of the registry.  The items that can be configured are listed below:

RuUO.EXE

To configure the path to the RunUO executable, click on the button to the right of the path field and an open file dialog will appear.  Browse to the location of the RunUO.exe and select the file and click OK.

Ultima Client

The path to the Ultima client is configured automatically for you by the application based on the registry keys used by the Ultima Online client.  This has been tested with all clients except for the Age Of Shadows client.  If the client can not be determined by
the application at startup, click on the button to the right of the path field and browse to the folder where the Ultima client is located.  Select the client.exe or uotd.exe and click OK.  This does two things.  First, is to provide the editor with the path to all of the neccessary client files, namely the map*.mul files.  The second is to set the default Client Window setting used when the Sync option is checked in the editor.  This is needed to send commands (keystrokes) to the client window if neccessary.

Default Zoom Level

This sets the default zoom level when the editor starts.

Command Prefix

This sets the default command prefix to be used when sending commands to the client window.

Client Window

The client window is used to set the default client window title of the Ultima client.  This is automatically set when the Ultima client path is selected of the client is either client.exe or uotd.exe.  If any other file is selected as the Ultima client, this value will default to ??? and will then have to be manually configured if the Sync option is to operate properly.

Default Spawn Details

The default values for a new spawner can be set here.  When the Restored Defaults button is pressed in the Spawn Details group located in the top right panel, these values will be loaded.

After all of these values have been set, the editor should be able to operate normally.  If any of these values need to be changed at a later date, click on the Setup button in the Map Settings group located in the top left panel of the editor.

Commands  [Top]

Since you probably have a few spawns defined already using the spawner that comes with RunUO, there are a few commands that will help to get those spawns into the editor.  The commands are defined in the SpawnEditorCommands.cs script and are as follows:

  • SpawnShowAll
    Shows all spawners by making them visible and changing their object id to that of a ships mast for easy visiblity.
    Use this command to make all of the spawners more visible.  Useful when trying to find spawners with incorrect Z values.
  • SpawnHideAll
    Hides all spawners by making them invisible and reverting them back to their normal object id.
    Use this command to undo the SpawnShowAll command.
  • SpawnWipe
    Wipes ALL spawners from the current map.
    The current map is based on the map where the mobile who issued the command is located.
  • SpawnSave <FileName>
    Saves all spawner information for the current map to the file name specified which can then be loaded in Spawn Editor.
    Creates an XML based file that the Spawn Editor and the XmlSpawner class can both use.
  • SpawnLoad <FileName>
    Loads all spawner information from the file specified (file created by the Spawn Editor or SpawnSave or XmlSpawnerSave command).
    Uses the XML based file that the Spawn Editor or the XmlSpawner class created to populate a world.
  • SpawnEditorGo <map> | <map> <x> <y> [<z>]
    Go command used with the Spawn Editor when the Sync option is checked, takes the name of the map as the first parameter.

 

The XmlSpawner.cs script contains all of the commands, gumps, and spawner code needed to use the XmlSpawner object that is designed to work with the Spawn Editor.  The commands used by the XmlSpawner object are as follows:

  • XmlSpawnerShowAll
    Makes all XmlSpawner objects visible and movable and also changes the item id to a blue ships mast for easy identification.
  • XmlSpawnerHideAll
    Makes all XmlSpawner objects invisible and unmovable and returns the object id to the default value.
  • XmlSpawnerWipe
    Removes all XmlSpawner objects from the current map.
  • XmlSpawnerWipeAll
    Removes all XmlSpawner objects from the entire world.
  • XmlSpawnerLoad
    Loads XmlSpawner objects into the world as defined in the file specified (file created by Spawn Editor or XmlSpawnerSave or SpawnSave).
  • XmlSpawnerSave
    Saves all XmlSpawner objects from the current map into the file specified.
  • XmlSpawnerSaveAll
    Saves ALL XmlSpawner objects from the entire world into the file specified.

How-To  [Top]

Below are some short tutorials on how to use the Spawn Editor to create an manipulate spawns.

Creating A Spawner

  1. Pick the desired map from the list of maps in the Map Settings area in the top left panel.  Lets pick Trammel.
  2. Left-Click on the map to move the map to the desired position.
  3. Zoom in/out of the map using the slider in the Map Settings area to make defining the spawn area easier.
  4. Left-Click in the top-left corner of the spawn area and drag the mouse down to the bottom-right corner of the spawn area.  A white rectangle should be displayed showing the area that the spawner will define.
  5. Use the Spawn Types list to select what type of mobiles and/or items you would like to spawn in this area.
  6. Give the spawner a name by changing the default name in the Spawn Details area in the top right panel.  Lets call it Trammel1.
  7. Set the rest of the spawner properties in the Spawn Details area as required.
  8. Right-Click within the white rectangle.  This will create the spawner and add it to the Spawn List in the bottom right panel.  The spawn rectangle should now change from white to light blue.
  9. Click on the Save button in the Spawn List and save it to a file called MySpawner.xml into your RunUO directory.
  10. Start RunUO (did you place the SpawnEditorCommands.cs and XmlSpawner.cs in the RunUO scripts directory? If not go back to the Installation section and complete this step).
  11. Login to RunUO with and administrator account.
  12. Load the spawn file just save using the command [XmlSpawnerLoad MySpawner.xml
  13. To see the spawner, go back to Spawn Editor and click on the Sync check box so that it has a check mark beside it.
  14. Click on the spawner you created in the Spawn List (Trammel1).
  15. You should be teleported to the exact location of the spawner.
